# Figure catalog

EDA / glass-characterization figures, all generated by `analysis/make_slides.py`
(regenerate with `python3 analysis/make_slides.py` from the repo root). Every EDA figure
uses **real labelled scans** — no synthetic stand-ins. Color convention is constant:
**blue = glass, gold = frame, green = ground, red = interior.**

| File | What it shows |
|------|---------------|
| `S03_ransac_single_board.png` | Real RANSAC detection on a single board: fitted plane (cyan), traced boundary (magenta), four extracted corners (yellow). The geometric baseline working on one board. |
| `S03_ransac_baseline.png` | Why one plane fails on a real facade: (left) it's a *grid of windows*, not one board; (right) top-down, frame / glass / interior sit at *different depths* a single plane can't capture. |
| `S06_pointnet_motivation.png` | PointNet++ set-abstraction schematic — sample centroids, group neighbours in a ball, pool to a feature — i.e. the density + geometry cues RANSAC hand-tuned, but learned. |
| `S07_testbed_schematic.png` | Schematic of the 6-DOF rig: known pose → ground-truth sensor position + window corners. |
| `S08_pointcloud_explainer.png` | A real raw scan coloured by intensity (the window grid is already visible) + a plain "what is a point cloud" panel. |
| `S10a_labels.png` | Labelled real scans (L6 / Oxy / Bech): glass / frame / interior / ground. |
| `S10b_class_distribution.png` | Per-point class mix per dataset (glass is the minority class). |
| `S10c_cloud3d.png` | One dense labelled L6 cloud in 3-D — the window grid is obvious. |
| `S11_sim_to_real_gap.png` | Simulator (dense, uniform, every beam returns) vs a real dense scan (patchy glass, holes, intensity clipped at 255). |
| `S12a_angular_glass.png` | Glass intensity vs incidence angle with 25–75% bands — L6 spike head-on, Oxy flat-bright, Bech flares to 255 at grazing. |
| `S12b_saturation_hist.png` | Saturation: (left) shape of the un-clipped returns; (right) % of glass pinned at 255 — L6 3% · Oxy 1% · Bech 87%. |
| `S12c_seethrough.png` | Top-down: beams pass *through* glass and return from inside the room (interior behind the glass plane). |
| `S13_forward_model.png` | Plain-language model: the three ways a surface returns the laser (matte / mirror / grazing), and the three real glasses as different mixes. |
| `S14_buildings_gallery.png` | Procedural building generator (facade systems + footprints) for synthetic training scenes. |
| `S16_results_placeholder.png` | Confusion-matrix + metrics template, marked PENDING INFERENCE. |
